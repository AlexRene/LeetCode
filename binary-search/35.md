Given a sorted array of distinct integers and a target value, return the index if the target is found. If not, return the index where it would be if it were inserted in order.

You must write an algorithm with **O(log n)** runtime complexity.

**Example 1:**

**Input:** nums = [1,3,5,6], target = 5
**Output:** 2

**Example 2:**

**Input:** nums = [1,3,5,6], target = 2
**Output:** 1

**Example 3:**

**Input:** nums = [1,3,5,6], target = 7
**Output:** 4

**Constraints:**

1 <= nums.length <= 104
-104 <= nums[i] <= 104
nums contains distinct values sorted in ascending order.
-104 <= target <= 104

#### Explicação

#### 1. Entendimento do problema

O objetivo é encontrar o índice de um valor (target) em um array ordenado.

Se o valor existir → retornamos seu índice
Se não existir → retornamos a posição onde ele deveria ser inserido mantendo a ordem

O algoritmo deve ter complexidade O(log n) → isso indica que devemos usar busca binária

#### 2. Ideia da solução

Aqui usaremos Binary Search (Busca Binária):

left → início do intervalo
right → fim do intervalo
meio → posição central do intervalo

A cada passo:

Comparamos nums[meio] com target
Eliminamos metade do array

#### 3. Como o algoritmo funciona
1. Começamos com:

    * left = 0
    * right = nums.length - 1
    * Enquanto left <= right:

    Calculamos o meio:

    * meio = left + (right - left) / 2
2. Comparações:

    * Se nums[meio] == target → encontramos o valor → retornamos meio

    Se nums[meio] < target → o target está à direita → movemos:

    * left = meio + 1

    Se nums[meio] > target → o target está à esquerda → movemos:

    * right = meio - 1
Quando o loop termina:

Significa que não encontramos o target
Nesse momento, left será exatamente a posição onde o elemento deve ser inserido

Por isso retornamos:

return left;

3. Intuição importante (ESSENCIAL)

    Quando o loop termina:

    right fica antes da posição correta
    left fica na posição correta de inserção

    Ou seja:

    left sempre aponta para o menor índice onde o target pode ser colocado sem quebrar a ordenação

4. Complexidade
    * Tempo: O(log n)
    * Espaço: O(1)

#### 4. Edge Cases
Target menor que todos os elementos → retorna 0

Target maior que todos os elementos → retorna nums.length

Array com 1 elemento → funciona normalmente

Target não existe no array → retorna posição correta de inserção
