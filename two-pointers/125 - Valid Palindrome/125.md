#### A phrase is a palindrome if, after converting all uppercase letters into lowercase letters and removing all non-alphanumeric characters, it reads the same forward and backward. Alphanumeric characters include letters and numbers.

_Given a string s, return true if it is a palindrome, or false otherwise._

**Example 1:**

**Input:** s = "A man, a plan, a canal: Panama"\
**Output:** true\
**Explanation:** "amanaplanacanalpanama" is a palindrome.

Example 2:

**Input:** s = "race a car"\
**Output:** false\
**Explanation:** "raceacar" is not a palindrome.\

Example 3:

**Input:** s = " "\
**Output:** true\
**Explanation:** s is an empty string "" after removing non-alphanumeric characters.
Since an empty string reads the same forward and backward, it is a palindrome.

**Constraints:**

1 <= s.length <= 2 * 105\
s consists only of printable ASCII characters.

---

### Explicação

#### 1. Entendimento do problema

O problema pede que verifiquemos se determinada cadeia de caracteres
é ou não um palindromo, considerando apenas caracteres alfanumericos
e letras minusculas, para resolver o problema primeiro precisamos de duas coisas:

* Garantir que todas as letras sejam minusculas
* Ignorar os espaços, pontuação, etc

#### 2. Ideia da solução

A abordagem escolhida foi usar o padrão Two Pointers, pois queremos comparar caracteres simétricos da string.

A ideia central é:

Usar dois ponteiros, um no início (left) e outro no final (right)
mover ambos em direção ao centro comparar apenas caracteres válidos (alfanuméricos)

#### 3. Como o algoritmo funciona

1. Inicializamos 2 ponteiros

* left = 0
* right = tamanho da string - 1

E então fazemos uma verificação inicial de um edge case

* Se s for uma string vazia retornamos true

2. Enquando left < right:
    * Primeiro ignoramos caracteres inválidos em ambos os ponteiros, fazemos isso pois se for o contrario podemos avançar quando deveriamos comparar, além de perder caracteres importantes
        * Se s[left] não for valido -> avançamos left
        * Se s[right] não for valido -> recuamos left
    * Depois comparamos os caracteres:
        * Convertemos tudo para minusculo
        * Se forem diferentes -> Retorna falso
    * Se forem iguais:
        * Avançamos left
        * Recuamos right
3.  Se o loop terminar sem encontrar diferenças
    * Retornamos true

4. Complexidade
    * Tempo O(n)
        Percorremos a string uma única vez
    * Espaço O(1)
        Não foram usadas estruturas auxiliares (in-place)

5. Edge Cases
    * String vazia → considerada palíndromo
    * String com apenas caracteres inválidos → também válida
    * Diferença entre maiúsculas e minúsculas → tratada com toLowerCase()
